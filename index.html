  2. Create accounts on GitHub and Cloudflare

üîç In-Depth Explanation

What is Node.js?

Node.js is a JavaScript runtime that lets you run JavaScript outside the browser. We need it to:

  • Run the development server on your computer
  • Use npm (Node Package Manager) to install tools
  • Build the website for production

To verify installation, open terminal and run:

node --version

You should see something like v20.x.x

Why GitHub?

GitHub is where your code lives online. It:

  • Backs up your code in the cloud
  • Tracks every change you make (version control)
  • Connects to Cloudflare for automatic deployments
  • Lets others fork/copy your project

Why Cloudflare Pages?

Cloudflare Pages hosts your website for free with enterprise-level features:

  • Global CDN (300+ locations worldwide)
  • Automatic HTTPS/SSL
  • Unlimited bandwidth
  • Auto-deploys when you push to GitHub

Step 3: Push to GitHub

‚ö° TL;DR

  1. Create a new repo on GitHub (don't initialize with README)
  2. Run these commands:
git remote set-url origin https://github.com/YOUR_USERNAME/YOUR_REPO.git git add . git commit -m "My personal website" git push -u origin main

üîç In-Depth Explanation

Creating Your GitHub Repository

  1. Go to github.com/new
  2. Name it something like my-website or portfolio
  3. Important: Do NOT check "Add a README" - we already have files
  4. Click "Create repository"

Understanding Git Commands

git remote set-url origin [URL]

Changes where your code will be pushed. The cloned repo points to the template‚Äîthis redirects it to YOUR repo.

git add .

Stages all changed files. The . means "everything in this folder." Git now knows which files to include in the next commit.

git commit -m "message"

Creates a snapshot of your staged files with a description. Think of it as a save point you can return to.

git push -u origin main

Uploads your commits to GitHub. -u origin main sets this as the default, so next time you can just type git push.

Future Changes

After the initial setup, updating your site is just:

git add . git commit -m "Updated about section" git push

üîç In-Depth Explanation

Connecting to Cloudflare Pages

  1. Open Cloudflare Dashboard

    Go to dash.cloudflare.com and log in

  2. Navigate to Pages

    In the sidebar, click "Workers & Pages"

  3. Create New Project

    Click "Create" ‚Üí "Pages" ‚Üí "Connect to Git"

  4. Authorize GitHub

    First time? You'll need to authorize Cloudflare to access your GitHub repos

  5. Select Your Repository

    Find and click on your website repo

  6. Configure Build Settings
    Setting Value
    Production branch main
    Build command npm run build
    Build output directory dist
  7. Deploy!

    Click "Save and Deploy" - takes about 30-60 seconds

üîß Troubleshooting

‚ùå "npm not found"

Fix: Node.js isn't installed or not in PATH. Reinstall from nodejs.org and restart your terminal.

‚ùå "git not found"

Fix: Install Git from git-scm.com

‚ùå Build fails on Cloudflare

Fix: Check build logs in Cloudflare dashboard. Common issues:

  • Wrong build command (should be npm run build)
  • Wrong output directory (should be dist)
  • Missing package.json

‚ùå Changes not showing on live site

Fix: Make sure you committed AND pushed:

git add . git commit -m "my changes" git push

‚ùå Port 3000 already in use

Fix: Another process is using port 3000. Either close it or run Vite on a different port:

npm run dev -- --port 3001
